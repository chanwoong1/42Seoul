# Introduction
## bash vs shell

**bash**

	bash는 GNU 운영체제을 위한 쉘 또는 명령어 인터프리터이다. 이름의 유래는 현재 Unix shell의 직계 조상인 'Stephen Bourne'의 말장난으로, 'Bourne-Again SHell'이며, Bell Labs Research 버전의 유닉스 7판에 등장했다.

	bash는 대부분 sh와 호환되며, ksh, csh의 유용한 기능을 통합하고 있다. IEEE POSIX 사양의 IEEE POSIX 쉘 및 도구 부분의 준수 구현을 위한 것이다. 대화형 및 프로그래밍 사용 모두에 대해 sh보다 기능 향상을 제공한다.

	GNU 운영체제는 csh를 포함한 다른 쉘들을 제공하지만, bash는 기본적인 shell이다. 다른 gnu 소프트웨어와 마찬가지로 bash는 이식성이 뛰어나다. 현재 거의 모든 버전의 유닉스와 몇 가지 다른 운영체제에서 실행된다.

**shell**

	기본적으로, 쉘은 명령어를 실행하기 위한 단순한 매크로 프로세서이다. 매크로 프로세서란 텍스트와 기호를 확장하여 더 큰 표현을 만드는 기능을 의미한다.

	유닉스 쉘은 명령어 인터프리터이자 프로그래밍 언어이다. 명령어 인터프리터로써, 쉘은 풍부한 GNU 유틸리티 세트에 대한 유저 인터페이스를 제공한다. 프로그래밍 언어 기능을 사용하면 이러한 유틸리티를 결합할 수 있다. 명령을 포함하는 파일들을 생성하고 명령 자체가 될 수 있다. 이 새로운 명령어들은 /bin과 같은 디렉토리에 있는 시스템 명령어들과 같은 상태를 가지며, 유저 또는 그룹이 사용자 지정 환경을 설정하여 공통 작업을 자동화 할 수 있다.

	쉘은 대화식 또는 비대화식으로 사용될 것이다. 대화형 모드에서는 키보드에서 입력한 것들을 받아들인다. 비대화식으로 실행할 때 쉘은 파일에서 읽은 명령들을 실행한다.

	쉘을 사용하면 동기 및 비동기로 GNU 명령을 실행할 수 있다. 쉘은 추가 입력을 받기 전에 동기 명령이 완료될 때까지 기다린다. 비동기 명령은 추가 명령을 읽고 실행하는 동안 쉘과 병렬로 계속 실행된다. 리다이렉션 구성을 사용하면 해당 명령의 입력 및 출력을 세밀하게 제어할 수 있다. 또한, 쉘은 명령 환경의 내용을 제어할 수 있다.

	쉘은 또한 별도의 유틸리티를 통해 얻을 수 없거나 불편한 기능을 구현하는 작은 기본 명령 세트를 제공한다. 예를 들어 cd, break, continue 및 exec는 쉘 자체를 직접 조작하기 때문에 쉘 외부에서 구현할 수 없다. history, getopts, kill, pwd 내장 기능 등은 별도의 유틸리티로 구현할 수 있지만, 내장 명령으로 사용하는 것이 더 편리하다.

	명령을 실행하는 것이 필수적이지만 쉘의 기능의 대부분은 포함된 프로그래밍 언어에서 나온다. 다른 고급 언어와 마찬가지로 쉘은 변수, 흐름 제어 구성, 인용 및 기능을 제공한다.

	쉘은 프로그래밍 언어를 보강하기 보다는 대화식 사용을 위해 특별히 설계된 기능을 제공한다. 이러한 대화형 기능에는 작업 제어, 명령줄 편집, 명령 기록 및 별칭이 포함된다.

# Definitions

이 정의들은 매뉴얼의 나머지 부분에서 사용된다.

POSIX

	유닉스를 표준 기반으로 한 오픈 시스템이다. 배쉬는 주로 포식스 표준의 쉘 및 유틸리티 부분과 관련이 있다.

BLANK

	공백 혹은 탭 문자이다.

BUILTIN

	파일 시스템의 어딘가에서 실행 가능한 프로그램이 아니라 쉘 자체에 의해 내부적으로 구현되는 명령이다.

CONTROL OPERATOR

	제어 기능을 수행하는 토큰이다. 개행 또는 '||', '&&', '&', ';', ';;', ';&', ';;&', '|', '|&', '\''.

EXIT STATUS

	명령 호출에 의해 반환되는 값. 이 값은 8비트로 제한되므로, 최대값은 255이다.

FIELD

	쉘 확장 중 하나의 결과인 텍스트의 단위. 확장 후, 명령을 실행할 때, 결과 필드는 명령 이름 및 인수로 사용된다.

FILENAME

	파일을 식별하는데 사용되는 문자열.

JOB

	파이프라인과 파이프라인에서 파생된 모든 프로세스로 구성되며 모두 동일한 프로세스 그룹에 있는 프로세스 집합이다.

JOB CONTROL

	사용자가 프로세스 실행을 선택적으로 중지 및 다시 시작할 수 있는 메커니즘이다.

METACHARACTER

	따옴표 없이 단어를 구분하는 문자. 메타 문자는 공백, 탭, 개행, 또는 '|', '&', ';', '\'', '<', '>' 문자 중 하나이다.

NAME

	문자, 숫자, 밑줄만으로 구성된 단어로 문자나 밑줄로 시작하는 단어이다. 이름은 쉘 변수 및 함수 이름으로 사용된다. 식별자라고도 한다.

OPERATOR

	제어 연산자 또는 리다이렉션 연산자이다. 리다이렉션 연산자 목록은 리다이렉션 연산자를 참조하라. 연산자에는 인용 부호가 없는 메타 문자가 하나 이상 포함되어 있다.

PROCESS GROUP

	동일한 프로세스 그룹 ID를 각각 같는 관련 프로세스의 모음이다.

PROCESS GROUP ID

	프로세스가 실행중인 동안 프로세스 그룹을 나타내는 고유 식별자이다.

RESERVE WORD 

	쉘에 특별한 의미가 있는 단어이다. 대부분의 예약어에는 for 및 while과 같은 쉘 흐름 제어 구조를 도입힌다.

RETURN STATUS

	종료 상태의 동의어이다.

SIGNAL

	시스템에서 발생하는 이벤트를 커널이 프로세스에 알릴 수 있는 메커니즘이다.

SPECIAL BUILTIN

	POSIX 표준에 의해 특별 분류된 쉘 내장 명령.

TOKEN

	쉘에서 단일 단위로 간주되는 일련의 문자이다. 단어 또는 연산자이다.

WORD

	쉘에서 하나의 단위로 취급되는 일련의 문자. 단어는 인용되지 않은 메타 문자를 포함할 수 없다.

# Basic Shell Features

## Shell Syntax

쉘은 입력을 읽을 때, 일련의 작업들을 진행한다. 주석이거나, 주석이 아닌 것으로 입력을 받는다.

받은 입력은 단어와 연산자로 나누고, 인용 규칙을 사용하여 다양한 단어와 문자를 할당한다.

### Shell Operation

	다음은 쉘이 명령어를 읽고 실행할 때의 동작에 대한 간략한 설명이다. 기본적으로 쉘은 다음을 수행한다.

	1. 인수로 제공된 문자열에서 파일의 입력을 읽는다.
	2. 따옴표의 규칙에 따라 입력을 단어와 연산자로 나눈다.
	3. 토큰을 단순 및 복합 명령으로 구문 분석한다.
	4. 다양한 쉘 확장을 수행하여 확장된 토큰을 파일 이름 목록과 명령 인수로 나눈다.
	5. 필요한 리다이렉션을 수행하고 인수 목록에서 리다이렉션 연산자와 해당 피연산자를 제거한다.
	6. 명령을 실행한다.
	7. 선택적으로 명령이 완료될 때까지 기다리고 종료 상태를 수집한다.

### Quoting

	따옴표는 특정 문자나 단어의 특별한 의미를 쉘로 제거하는 데 사용된다. 따옴표는 특수 문자에 대한 특수 처리를 비활성화 하고 예약어가 그렇게 인식되는 것을 방지하고 매개변수 확장을 방지하는 데 사용할 수 있다.

#### Escape Character

	인용 부호 없는 백슬래시는 탈출문자이다.

#### Single Quotes

	작은따옴표 안에 있는 각 문자의 리터럴 값을 유지한다. 앞에 탈출문자가 있더라도 작은따옴표 사이에는 작은따옴표가 올 수 없다.

#### Double Quotes

	큰따옴표는 작은따옴표를 제외하고 따옴표 안에 있는 모든 문자의 리터럴 값을 유지한다.

#### ANSI-C Quoting

	\a	경고(벨)

	\b	역행 키이

	\e
	\E	이스케이프 문자(ANSI C 아님)

	\f	양식 피드

	\n	개행

	\r	캐리지 리턴

	\t	가로 탭

	\v	세로 탭

	\\	백슬래시
	
	\'	작은따옴표

	\"	큰따옴표

	\?	물음표

	\nnn	값이 8진수 값인 8비트 문자 nnn (1~3개의 8진수)
	
	\xHH	값이 16진수 값 HH (16진수 또는 2자리) 인 8비트 문자

	\uHHHH	값이 16진수 값 HHHH (1-4개의 16진수) 인 유니코드(ISO/IEC 10646) 문자

	\UHHHHHHHH	값이 16진수 값인 유니코드(ISO/IEC 10646) 문자 HHHHHHHH (1~8개의 16진수)
	
	\cx	컨트롤 - x 문자
